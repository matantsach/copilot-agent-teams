Lead Reads the Plan & Creates Tasks
The team-lead agent reads your plan file, understands the architecture, then creates tasks directly on the board using create_task. No GitHub needed.

Teammates Work in Parallel
Each teammate registers, claims a task, does the work, and submits. The claimed_at timestamp tracks duration.
1

2

3
Edit files, run tests in isolated worktree
4
update_task(status: "completed", result: "...")
!

T + 0:12
Review Checkpoint
Tasks in needs_review wait for the lead (or you) to approve or reject. Rejected tasks get sent back with feedback.

Approve

approve_task(task_id: 1)

Task → completed

Dependent tasks auto-unblock

Reject

reject_task(task_id: 1, feedback: "Missing rate limiting")

Task → in_progress

Feedback sent as message to teammate

After all 3 tasks are approved, task #4 auto-unblocks:
#1 Payment routing & middleware completed teammate-1
#2 Stripe billing integration completed teammate-2
#3 Billing dashboard UI completed teammate-3
#4 Integration tests pending ← unblocked!
